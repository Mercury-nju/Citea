## 🎯 需求

- ✅ 后台系统独立部署（AWS/其他平台）

- ✅ 通过 API 或共享数据库连接
- ✅ 不影响主网站

## 🏗️ 架构方案

### 方案 A: 独立 Next.js 应用（推荐）

```
主网站（Vercel）         后台系统（AWS）
    ↓                       ↓
  用户端功能              管理员界面
    ↓                       ↓
共享 Redis 数据库 ←─────→ 通过 REDIS_URL 连接
```

**优点**：
- ✅ 完全独立部署
- ✅ 可以独立维护
- ✅ 不影响主网站性能
- ✅ 可以使用不同的技术栈

### 方案 B: 独立前端 + API

```
后台前端（AWS S3 + CloudFront）

```
